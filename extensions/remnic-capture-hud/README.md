# remnic-capture-hud

Highlights in the pi TUI whenever **Remnic** captures a memory during a session.

## What it does

- Watches the Remnic memory dir (`~/.remnic/memory`, or `remnic.memoryDir` from
  `~/.config/remnic/config.json`) recursively.
- When an extraction pass writes new memory files, shows:
  - a **toast** — single capture, or a batched summary (`captured 3 memories (2 questions, 1 decision)`);
  - a **widget** above the editor — running list of this session's captures, newest first;
  - a compact **footer status** — `🧠 remnic: N`.
- Namespace-aware: catches both the default namespace and project namespaces
  under `namespaces/<ns>/…`, tagging non-default captures.

## Why it's a separate extension

The `extensions/remnic/` dir is generated by `remnic connectors install pi` and
gets overwritten on remnic upgrades. This companion lives on its own so it
survives upgrades.

## Latency

None on the turn path. Remnic's extraction already runs asynchronously inside the
daemon (~15-20s after a turn); this extension only reacts to the filesystem event
when a memory file lands. It never blocks input, the model response, or the turn.

## Config (optional, env)

| Env | Default | Purpose |
| --- | --- | --- |
| `REMNIC_MEMORY_DIR` | from daemon config, else `~/.remnic/memory` | dir to watch |
| `REMNIC_HUD_MAX_LIST` | `6` | widget rows |
| `REMNIC_HUD_CATEGORIES` | `decisions,facts,questions,entities,corrections,preferences` | categories to surface |

`artifacts` are intentionally excluded (Remnic dual-writes each decision as an
artifact mirror, so including them would double every hit).

## Activate

Auto-discovered at `~/.pi/agent/extensions/remnic-capture-hud/index.ts`. Run
`/reload` or start a new pi session to load it.
